Model Submission Tutorial in R

An Allstate Data Scientist

2/15/2017

The scope of this tutorial

Building a good predictive model is equal parts science and art, and this tutorial will disappoint on both fronts. It will do so by not only leaving out the very basic things that need to be done to construct a model you can trust (things like appropriately validating your model), but also by not doing things that need to be done to construct a good model (meaningful features, parameter tuning).

Think of this tutorial as the shortest path from the input file that has been provided for the hackathon, to a file that you can submit to the leaderboard and get a score back. This tutorial is a starting point; it’s mission is to clarify some aspects of the data that may otherwise be less than obvious.

Building the submission model

Let’s load some of the libraries that we will need for this tutorial. If you don’t have one or more of these libraries installed, take a look at the install.packages function in R.

library(data.table) # tutorial assumes  v1.9.7
library(gbm) # CRAN v2.1.1

Once the libraries are loaded, let’s load the data. I have assumed that the data is present inside the data directory in the same folder as this code. If you download and save the data elsewhere, you will likely need to replace ./data/train.csv by /path/to/your/data/train.csv.

data_all <- fread('./data/train.csv')
ids_test <- fread('./data/test.csv')

In order to create a model, I will need to count how many times each event has occurred on a given id. The next piece of code creates a dummy counter to assist with that task.

data_all[, dummy_counter:=1]

The next few sections walk through the bulk of the model building exercise.

1. Find the first/last event for each policy in training data

data_train <- subset(data_all, !(id %in% ids_test$id))

# find the first/last event for each policy in training data
data_train[, max_timestamp_by_policy:=max(timestamp), by=id]
data_train[, min_timestamp_by_policy:=min(timestamp), by=id]

2. Create the target variable for the model

To do this, we create a dataset that contains the last event for each policy. This will get merged onto the training frame we create by the id.

data_train_last_element = subset(data_train, timestamp == max_timestamp_by_policy)
data_train_last_element$timestamp = NULL
data_train_last_element$dummy_counter = NULL
data_train_last_element$max_timestamp_by_policy = NULL
data_train_last_element$min_timestamp_by_policy = NULL
setnames(data_train_last_element, "event", "last_event")

3. Create some features for the model

One possible feature is the first event in the sequence per id. To create this feature, we pick the first event per id, isolate that in a separate data table, and join that back onto our final frame by id.

data_train_first_element = subset(data_train, timestamp==min_timestamp_by_policy)
data_train_first_element$timestamp = NULL
data_train_first_element$dummy_counter = NULL
data_train_first_element$max_timestamp_by_policy = NULL
data_train_first_element$min_timestamp_by_policy = NULL
setnames(data_train_first_element, "event", "first_event")

The next feature to create is the count of how many times an event has happened on an id. This is where our dummy_counter variable comes in handy. One thing to note: the last event on a policy on the training data is the target variable, and shouldn’t be included in the counts.

# featurize data, without the last element, by looking at counts of events
data_train_allButLast <- subset(data_train, timestamp != max_timestamp_by_policy)
data_train_ohe <- dcast(data_train_allButLast, 
                        id ~ event, 
                        fun.aggregate=sum, 
                        value.var="dummy_counter")

Yet another feature is the total number of events that occurred (not including, obviously, the last event).

# create a feature that counts how many events occured in total before the last event
data_train_ohe[, total_events:=apply(data_train_ohe[, -c("id"), with=FALSE], 1, sum)]

data_train_ohe <- merge(data_train_ohe, data_train_first_element, by=c("id"))

data_train_merged <- merge(data_train_ohe, data_train_last_element, by=c("id"))

4. Training a model

Ok, so things looks good so far. Let’s train a basic model.

# train a gbm model
# WARNING: Run time can exceed 10-15 minutes. For faster run times, lower n.trees=10
model <- gbm(last_event ~ . - id, 
             distribution="multinomial", 
             data=data_train_merged, 
             interaction.depth=5,
             n.trees=100,
             shrinkage=0.05,
             train.fraction=0.8,
             verbose=TRUE)

Scoring the model on the test data

We’ve build a model. To get something we can submit to the leaderboard, we just need to score this on the test data. To do that, we will re-do some of the main steps we performed with the training data.

  • Filter data_all to isolate the test data records
  • Create the first_event feature
  • Create the counts for each event code
  • Merge all of that together

Note that we don’t have to do the work with the last_event feature, because we are now working with the test set and we want to predict the last event. In order to score the model, we don’t need to know the last event.

# create the test dataset
data_test <- subset(data_all, id %in% ids_test$id)
data_test[, min_timestamp_by_policy:=min(timestamp), by=id]

# create the first event feature
data_test_first_event <- subset(data_test, timestamp==min_timestamp_by_policy)
data_test_first_event$timestamp = NULL
data_test_first_event$dummy_counter = NULL
data_test_first_event$min_timestamp_by_policy = NULL
setnames(data_test_first_event, "event", "first_event")

data_test_ohe <- dcast(data_test, id~event, fun.aggregate=sum, value.var="dummy_counter")
# add the total events feature
data_test_ohe[, total_events:=apply(data_test_ohe[, -c("id"), with=FALSE], 1, sum)]

# add the first event feature
data_test_ohe <- merge(data_test_ohe, data_test_first_event, by=c("id"))

Finally, we construct the predictions variable which will contain the data that we want to write out to disk, and submit to the leaderboard.

# score the model
predictions_raw <- predict(model, data_test_ohe, type="response")
## Using 100 trees...
#package it into a format that can be scored
predictions <- as.data.table(predictions_raw[,,1])
setnames(predictions, names(predictions), colnames(predictions_raw))
predictions[, id:=data_test_ohe$id]
